Alloy Editor

Android Studio wasn't built for FTC. So I built something that was.

GitHub — Coming Soon

The wrong tool for the job

After competing in FTC (FIRST Tech Challenge) robotics competitions, I got fed up. Android Studio is a mobile app IDE. It was designed to build apps for phones — not to program a competition robot under time pressure. It's bloated, it crashes for no reason, and the setup process alone can eat an entire practice session.

FTC teams across the world use it anyway because there's nothing else purpose-built for this workflow. The barrier to entry is real — new team members stall on IDE setup before they've written a single line of robot code.

Alloy Editor is my answer to this. Lightweight, stable, built specifically for the way FTC programmers actually work. Not a general-purpose IDE with FTC features bolted on — a tool that starts from the FTC workflow and builds outward.

"The right tool makes all the difference. FTC teams deserved one."

This project was born from frustration — the best kind of motivation to build something.

What Alloy Editor is built for

Lightweight by design

No unnecessary bloat. Starts fast, stays fast. Designed for the hardware reality of a student competition team — not a developer workstation.

FTC-native workflow

The editor understands FTC OpModes, hardware maps, and the development patterns that actually matter in competition robotics.

Built for stability

Android Studio crashes for no reason at the worst times. Alloy Editor is built to not do that. Stability is a feature.

From frustration to tool

Built by someone who lived the problem. Every design decision comes from real FTC competition experience — not spec documents.
