Rift

The mod loader that doesn't punish you for someone else's mistake.

Download Rift ↗ GitHub ↗

The problem with mod loaders

Every Minecraft mod loader loads mods as single units. If anything inside a mod throws an exception, the whole thing crashes. You get a crash report and spend time figuring out which of your mods is responsible.

Rift loads each mod's features independently. If one feature breaks, only that feature stops — the rest of the mod keeps running. It's still beta, and there's plenty to work on, but the core idea holds up.

The Rift API also abstracts away Minecraft version specifics, so mods don't need to be rewritten for each update. And it's the runtime that modrift-built mods target — the two projects are built to work together.

Key innovations

Feature-level fault isolation

Every mod feature is loaded as an isolated unit. One feature crashes? It fails gracefully. The rest of the mod — and every other mod — keeps running without interruption.

Version-agnostic mod API

The Rift API abstracts away Minecraft version specifics. Mods written for Rift work across versions. No rewrites, no compatibility matrices, no waiting for loader updates.

Open source (MIT)

Fully open source under the MIT license. The architecture is transparent — read it, fork it, contribute. Built in public because that's how trust is earned.

modrift ecosystem

Rift is the deployment target for mods built on modrift.dev. Build in the browser, ship to Rift. The two platforms are designed to work together end-to-end.
